# SoloMate 成员 B 开发方案：前端体验与旅行搭子内容负责人

## 1. 你的角色定位

成员 B 负责 SoloMate 的前端体验、搭子人格、任务内容、视觉风格和路演表达。

一句话：

> 你负责让 SoloMate 像一个真正的 AI 旅行搭子，而不是一个普通聊天机器人。

你的工作不是只做“美化”，而是把产品的陪伴感、趣味性和交互流程做出来。

---

## 2. 你的核心目标

用户一打开 App，就应该感受到：

```text
这是一个陪我旅行的 AI 搭子；
它可以听我说话；
它可以看我拍的照片；
它可以帮我做选择；
它可以用不同人格陪我；
它可以把旅行变成轻量任务；
它可以帮我留下旅行记忆。
```

## 5. 页面一：首页

### 目标

让用户立刻明白：

```text
SoloMate 是一个单人旅行陪伴 App。
用户可以马上开始语音、拍照或进入任务。
```

### 页面内容

```text
SoloMate
今天我陪你走。

选择一个旅行搭子，告诉我你现在的状态。
```

### 推荐按钮

```text
开始语音陪伴
拍照给搭子看
进入城市任务
生成今日旅行日记
```

### 首页组件

```text
当前搭子头像
欢迎语
开始语音按钮
拍照分享按钮
附近推荐卡片
今日状态卡片
```

### 示例文案

```text
今天我陪你走。
说一句现在的状态，我来帮你决定下一步。
```

## 6. 页面二：人格选择页

### 目标

人格选择不能只是换头像，而要让用户觉得自己真的在选择一个“搭子”。

### 必做人格

| 人格 | 用途 | 适合场景 |
|---|---|---|
| 温柔朋友型 | 陪聊、安慰、提醒安全 | 一个人出行、不安、迷茫 |
| 本地导游型 | 景点讲解、文化故事 | 想了解当地历史和文化 |
| 摄影搭子型 | 构图建议、拍照文案 | 想拍照、发朋友圈 |
| 省钱规划型 | 控制预算、推荐省钱路线 | 预算有限、不想花太多 |
| 游戏精灵型 | 发布任务、解锁徽章 | 想让旅行更有趣 |

### 每张人格卡片需要包含

```text
头像
名称
一句话介绍
适合场景
选择按钮
开场白
```

### 示例

```text
温柔朋友型
一句话：一个会安慰你、陪你走路、提醒你安全的朋友。
适合：一个人出行、不安、迷茫、需要陪聊。
开场白：今天我陪你走，不用急，我们慢慢来。
```

## 7. 页面三：语音对话页

### 目标

语音页要像“有人陪你走路”，不是普通聊天页面。

### 页面元素

```text
当前搭子头像
语音波纹按钮
用户语音转文字区域
AI 回复气泡
下一步建议按钮
安全提醒卡片
附近推荐卡片
```

### 回复展示原则

AI 回复不要太长，建议分成三段：

```text
先回应情绪
再给判断
最后给可执行选项
```

### 示例回复卡片

```text
我在，不用急。

你现在一个人出行，又有点累，我建议先去人多一点的夜市。

你可以选择：
去夜市 / 找咖啡店休息 / 回酒店
```

### 你需要配合成员 A 的字段

对话接口会返回：

```json
{
  "reply_text": "",
  "reply_type": "",
  "emotion_detected": "",
  "suggested_action": "",
  "safety_tip": "",
  "next_options": [],
  "task_triggered": ""
}
```

你的页面应展示：

```text
reply_text → 主回复气泡
safety_tip → 安全提醒卡片
next_options → 三个下一步按钮
task_triggered → 任务卡片弹出
emotion_detected → 当前心情标签
```

## 8. 页面四：照片分享页

### 目标

让用户感受到：

> 我把我看到的世界分享给 AI，AI 再用搭子的方式回应我。

### 页面流程

```text
上传/拍摄照片
→ 显示“搭子正在看你看到的世界”
→ 展示场景理解
→ 展示安全提醒/拍照建议
→ 如果完成任务，展示徽章动画
```

### 文案示例

```text
我看到了你眼前的街道。
这里有灯光、店铺和人流，看起来比较适合短暂停留。
你拍到了城市的烟火气，任务完成。
```

### 你需要展示的字段

图片分析接口会返回：

```json
{
  "scene_summary": "",
  "safety_observation": "",
  "photo_advice": "",
  "task_result": {
    "passed": true,
    "reward_badge": "",
    "reason": ""
  },
  "reply_text": ""
}
```

页面映射：

```text
scene_summary → 场景理解卡片
safety_observation → 安全观察卡片
photo_advice → 拍照建议卡片
task_result.passed → 任务完成状态
task_result.reward_badge → 徽章弹窗
reply_text → 搭子回应气泡
```

## 9. 页面五：城市任务与徽章页

### 目标

轻量游戏化，不做复杂游戏，但要有探索和奖励感。

### 页面内容

```text
当前任务
任务要求
上传照片按钮
完成状态
已获得徽章
未解锁徽章
```

### 推荐任务

| 任务名 | 完成方式 | 奖励 |
|---|---|---|
| 烟火气打卡 | 拍一张有本地生活气息的照片 | 城市烟火徽章 |
| 本地味道挑战 | 拍特色食物/菜单 | 本地味道徽章 |
| 安心路线选择 | 选择人多明亮路线 | 安心探索徽章 |
| 旅行第一步 | 完成第一次语音对话 | 不孤单徽章 |

### 徽章风格建议

```text
城市纪念章风格；
不要太幼稚；
可以用暖色、圆形、轻微发光；
任务完成时有简单弹出动画。
```

## 10. 页面六：旅行日记页

### 目标

让用户感受到“这趟旅行被记录了”，形成完整闭环。

### 页面内容

```text
今日走过的地点
完成的任务
获得的徽章
心情变化
AI 旅行日记
朋友圈文案
```

### 页面标题建议

```text
今天，一个人也走得很好
```

### 日记模板

#### 温柔日记版

```text
今天你一个人走过了 {places}。一开始你有点 {mood_start}，但后来你完成了 {task}，也慢慢放松下来。你解锁了 {badge}。这不是一次孤单的旅行，而是一次属于自己的探索。
```

#### 朋友圈文案版

```text
一个人的旅行，也会遇到刚刚好的热闹。
今天解锁：{badge}
地点：{places}
```

#### 路线总结版

```text
今日路线：{places}
完成任务：{tasks}
获得徽章：{badges}
心情变化：{mood_start} → {mood_end}
```

## 15. 路演 Demo 脚本

你负责把项目讲成一个故事。

### 开场

```text
很多人不是不想旅行，而是朋友没时间陪。
一个人旅行时，最缺的不是攻略，而是有人一起商量、有人陪你说话、有人提醒你安全、有人分享你看到的风景。
所以我们做了 SoloMate，一个 AI 旅行搭子。
```

### 演示流程

```text
1. 选择温柔朋友型搭子
2. 用户说：朋友临时来不了了，我现在一个人有点不知道去哪
3. AI 陪伴并推荐夜市
4. 用户上传夜市照片
5. AI 判断完成烟火气任务
6. 解锁城市烟火徽章
7. 生成今日旅行日记
```

### 收尾

```text
SoloMate 不是一个冷冰冰的旅行规划工具，而是一个能听你说、能看你看到的世界、能陪你做决定、还能帮你留下回忆的 AI 旅行搭子。
```
