# SoloMate 全面详细方案

## 1. 项目定位

**SoloMate：一个面向单人出行者的 AI 旅行搭子。**

SoloMate 不是普通旅行攻略生成器，而是一个场景化 AI Agent。它通过语音交互、位置感知、照片分享、搭子人格定制、轻量任务和旅行记忆，在朋友无法陪同出行时，为用户提供陪聊、路线决策、景点讲解、拍照建议、安全提醒和旅行记录。

### 一句话介绍

> 当朋友不能陪你旅行时，SoloMate 像一个 AI 搭子一样，通过语音、照片和位置感知陪你边走边聊、边看边决策，让一个人的旅行也有被陪伴的感觉。

---

## 2. 黑客松目标

评审标准主要是：

1. **技术实现**
2. **用户交互**
3. **创新性**

SoloMate 对应的展示重点：

| 评审维度 | SoloMate 对应亮点 |
|---|---|
| 技术实现 | LLM Agent、语音输入/播报、图片理解、位置感知、状态管理 |
| 用户交互 | 语音陪伴、人格选择、拍照分享、任务打卡、徽章反馈 |
| 创新性 | 单人旅行陪伴 + 情绪感知 + 场景化 Agent + 游戏化探索 |

黑客松不建议追求完整商业级旅行 App，而是完成一个稳定、清晰、有感染力的 MVP 闭环。

---

## 3. MVP 核心闭环

推荐只围绕下面这一条主线开发：

```text
用户打开 App
→ 选择旅行搭子人格
→ 语音说出当前状态
→ AI 根据位置、时间、心情给建议
→ 用户拍照分享当前场景
→ AI 看图分析并给反馈
→ 触发任务/解锁徽章
→ 生成旅行日记
```

这条线可以完整展示“陪伴、决策、看图、任务、记忆”五个核心价值。

## 4. 必做功能

### 4.1 搭子人格选择

用户进入 App 后选择不同陪伴人格。人格不是纯装饰，而是影响 AI 语气、建议策略和页面呈现。

| 人格 | 作用 | 典型风格 |
|---|---|---|
| 温柔朋友型 | 安慰、陪聊、提醒安全 | 温柔、自然、像朋友 |
| 本地导游型 | 景点讲解、文化故事、路线推荐 | 专业、亲切、简洁 |
| 摄影搭子型 | 拍照建议、构图建议、文案生成 | 审美、鼓励、轻松 |
| 省钱规划型 | 控制预算、推荐低成本路线 | 实用、清楚、少废话 |
| 游戏精灵型 | 发布任务、奖励徽章、探索引导 | 活泼、有游戏感 |

人格通过 `personas.json` 维护，由成员 B 设计内容，成员 A 在 Agent Prompt 中读取使用。

---

### 4.2 语音旅行搭子

用户可以通过语音和 AI 对话。

典型使用场景：

```text
用户：我朋友临时来不了了，我现在一个人在外面，有点不知道去哪。
AI：没关系，我陪你。你现在一个人出行，我建议先去人多、灯光亮的夜市街区，既能吃点东西，也更安心。
```

MVP 技术路线：

```text
点击麦克风
→ 浏览器语音识别为文本
→ 调用 /api/chat
→ 返回 AI 回复
→ 前端 TTS 播放
```

必须保留文本输入框，避免现场语音识别失效。

---

### 4.3 位置感知与路线决策

系统需要知道用户大致位置、当前时间、出行状态和附近地点。

黑客松阶段可以先用模拟地点数据，不强依赖真实地图 API。

示例地点数据：

```json
{
  "id": "night_market",
  "name": "夜市街区",
  "type": "food",
  "distance": 600,
  "safety_level": "high",
  "tags": ["吃饭", "人多", "夜间安全", "烟火气"],
  "description": "适合单人旅行者吃饭和短暂停留，人流较多，夜间相对安心。",
  "task_id": "firework_photo_task"
}
```

AI 根据用户状态给出建议：

```text
推荐：夜市街区
原因：
1. 你现在一个人出行；
2. 当前时间偏晚；
3. 夜市人流较多，相对安心；
4. 可以顺便完成“烟火气打卡”任务。
```

---

### 4.4 照片分享与 AI 看图分析

用户可以拍照或上传照片，让 AI 分析当前场景。

可支持的场景：

| 场景 | AI 输出 |
|---|---|
| 街道照片 | 判断环境特点，给安全提醒 |
| 景点照片 | 讲解景点，推荐拍照角度 |
| 餐厅/菜单照片 | 推荐吃什么 |
| 风景照片 | 生成朋友圈文案 |
| 任务照片 | 判断是否完成打卡 |

照片分析返回结构建议：

```json
{
  "scene_summary": "画面中有店铺、灯光和街道人流，接近夜市或商业街场景。",
  "safety_observation": "画面中灯光较充足，人流较多，单人短暂停留相对更安心。",
  "photo_advice": "可以把招牌和街灯放在画面上方，让烟火气更明显。",
  "task_result": {
    "passed": true,
    "reward_badge": "城市烟火徽章",
    "reason": "照片包含店铺、灯光和生活街区元素。"
  },
  "reply_text": "你拍到了很有生活气息的一面，烟火气打卡完成！"
}
```

### 4.5 旅行选择困难终结器

单人旅行时常见痛点不是没有攻略，而是不知道下一步该怎么选。

用户可以问：

```text
我现在有点累，但又不想浪费时间，附近有夜市、江边和博物馆，我该去哪？
```

AI 输出：

```text
推荐：夜市

原因：
1. 你现在有点累，需要补充体力；
2. 夜市人流较多，单人出行更安心；
3. 可以顺便完成“烟火气打卡”任务；
4. 比江边步道更适合夜间单人前往。

不建议：江边步道
原因：夜间人可能较少，不如商业区域安全。
```

---

### 4.6 安心守护模式

当用户表达不安、害怕、迷路、犹豫时，系统进入安心模式。

触发语句示例：

```text
我一个人有点害怕。
这条路好黑。
我不知道该不该继续往前走。
```

AI 回复示例：

```text
我在陪你。建议你先不要继续往更暗的地方走，优先返回刚才人多、灯光亮的主路。你可以打开导航去最近的便利店、地铁站或商场。我也可以帮你生成一条位置分享消息发给朋友。
```

注意边界：

> 这是辅助性安全提醒，不替代报警、急救或专业安全服务。

---

### 4.7 游戏化任务与城市徽章

轻量实现即可，不做复杂宝可梦式游戏。

示例任务：

```text
任务：烟火气打卡
要求：拍一张有本地生活气息的照片。
完成条件：照片中包含店铺、灯光、人流、招牌、小吃摊等元素之一。
奖励：城市烟火徽章。
```

任务流程：

```text
AI 发布任务
→ 用户拍照
→ AI 看图判断
→ 成功后解锁徽章
→ 推荐下一站
```

建议首版任务：

| 任务名 | 场景 | 完成方式 | 奖励 |
|---|---|---|---|
| 烟火气打卡 | 夜市/老街 | 拍生活气息照片 | 城市烟火徽章 |
| 本地味道挑战 | 餐饮街 | 拍本地食物 | 本地味道徽章 |
| 安心路线选择 | 夜间出行 | 选择人多明亮路线 | 安心探索徽章 |
| 旅行第一步 | 第一次使用 | 完成第一次语音对话 | 不孤单徽章 |

---

### 4.8 旅行日记生成

结束时，AI 根据对话、地点、照片、任务和心情变化生成旅行记录。

示例：

```text
今天你一个人从老街走到了夜市。刚开始你有点犹豫，也有点不安，但后来你完成了“烟火气打卡”任务，拍下了一张很有生活气息的照片。你解锁了“城市烟火徽章”，也证明了一个人的旅行并不孤单。
```

可以支持三种输出：

| 版本 | 用途 |
|---|---|
| 温柔日记版 | 记录心情 |
| 朋友圈文案版 | 社交分享 |
| 路线总结版 | 复盘今天走过哪里 |

## 5. 非主线功能处理

### 5.1 桌面悬浮图标

建议只做 App 内悬浮按钮，不作为主线。

```text
App 内右下角固定一个搭子头像按钮
点击后弹出语音对话面板
用户切换页面时头像仍然悬浮在界面上
```

不建议黑客松阶段做 Android/iOS 系统级悬浮窗。

---

### 5.2 视频交互 / 伪 AR

建议作为展示功能，而不是主线功能。

推荐实现：

```text
打开摄像头
→ 屏幕叠加一个 2D 虚拟搭子/城市精灵
→ 用户点击角色触发任务
→ 用户拍照上传
→ AI 判断任务是否完成
```

不建议做真正实时视频理解和完整 AR 捕捉，风险过高。

## 11. 推荐 Demo 流程

```text
1. 用户打开 SoloMate
2. 选择“温柔朋友型”
3. 用户说：朋友临时来不了了，我现在一个人有点不知道去哪
4. AI 回复：我陪你，并推荐夜市
5. 页面展示附近地点：夜市、老街、河边步道
6. 用户切换到“游戏精灵型”
7. AI 发布“烟火气打卡”任务
8. 用户上传一张夜市/街道照片
9. AI 分析照片并判断任务完成
10. 解锁“城市烟火徽章”
11. 用户点击“生成今日旅行日记”
12. AI 生成一段温柔旅行记录
```

## 12. 同步规则

两个人必须共同遵守：

```text
1. 所有功能围绕同一条 Demo 流程开发。
2. 成员 A 不随意改接口字段，成员 B 不随意新增无法实现的交互。
3. personas.json、tasks.json、mock_places.json 是双方共同对齐的核心文件。
4. 每完成一个模块就立刻集成，不要最后一天才合并。
5. 真实 API 不稳定时，必须有模拟数据兜底。
```

最推荐协作方式：

```text
成员 B 先用假数据把前端流程做出来；
成员 A 同时把真实接口做出来；
最后把假数据替换成真实接口。
```

## 13. 最终总结

SoloMate 最重要的创新点不是“生成攻略”，而是“在单人旅行中提供陪伴式决策”。

最终要让评委感受到：

```text
它能听用户说话；
它能看用户看到的世界；
它能理解用户当前心情；
它能帮用户做下一步决策；
它能把旅行变成轻量游戏；
它能在结束后帮用户留下回忆。
```
